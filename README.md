## 🚀 Key Technological Features

### 1. Advanced Machine Learning Pipeline
* Ensemble Predictive Intelligence: Leverages multi-horizon regularized XGBoost Classifiers optimized for 1H, 4H, 12H, and 24H operational horizons.
* Feature Engineering: Computes over 10+ concurrency-driven mathematical indicators including Stochastic RSI, MACD, Bollinger Bands, and Average True Range (ATR) for volatility mapping.

### 2. Algorithmic Risk & Capital Management
* Dynamic Volatility Target: Automatically computes highly precise Stop-Loss (SL) and Take-Profit (TP) boundaries utilizing dynamic ATR scaling ratios.
* Capital Sizing: Implements rigorous mathematical volume scaling based on a maximum 2% total equity risk rule per directional trade.

### 3. Streamlined Architecture & Live Syncing
* Asynchronous Inference Handler: Offloads heavy mathematical CPU processes and prediction matrix generation to background worker threads via Python's asyncio pool to ensure zero-latency UI performance.
* Cross-Asset Correlation: Maps live assets against a static Bitcoin Dependency Coefficient Matrix (BTC_DEP) to continuously track structural market exposure.

## 🛠️ Installation & Deployment

### 1. Clone the Repository
```bash
git clone [https://github.com/parsaBABALOO/PBX-CRYPTO-TELEGRAM-BOT.git](https://github.com/parsaBABALOO/PBX-CRYPTO-TELEGRAM-BOT.git)
cd PBX-CRYPTO-TELEGRAM-BOT
```
### 2. Environment Ingestion & Dependency Installation
​Install all fundamental quantitative computing and interface libraries required for execution:
```bash
pip install python-telegram-bot ta yfinance pandas xgboost joblib
```
### 3. Authentication & Telegram Bot Token Configuration
​Before initiating the network engine, you must map your active Telegram Bot API Credentials into the system architecture. You can orchestrate this configuration through two distinct methodologies:
​Option A: Direct Assignment (Local Configuration)
​Open config.py and assign your custom HTTP API Token generated by @BotFather to the master initialization variable:
```bash
# Inside config.py
BOT_TOKEN = "YOUR_TELEGRAM_BOT_TOKEN_HERE"  # <-- Replace with your actual live token
```

Option B: Secure Environment Variable Injection (Production Standard)
​For high-security production deployments, leave config.py untouched and export your gateway token straight to the local environment pool before running:
Linux/macOS:
```bash
export TELEGRAM_BOT_TOKEN="your_actual_bot_token_here"
```
Windows (PowerShell):
```bash
$env:TELEGRAM_BOT_TOKEN="your_actual_bot_token_here"
```
### 4. Master Core Execution
​Once authentication variables are successfully initialized, boot up the high-level listener network gateway and begin live polling streams:
```bash
python main.py
```
